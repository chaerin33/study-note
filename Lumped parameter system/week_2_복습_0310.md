## 1. Parameter vs Property — 용어 정의

**개념 제시**

$$m\ddot{x} + c\dot{x} + kx = F(t)$$

위 ODE에서 $m$, $c$, $k$를 보는 관점은 두 가지다.

- **Property (속성/속수):** 물리 시스템 고유의 물성값. 스프링에 힘을 가해 변위를 측정하면 $F = k \cdot \delta x$ 관계로 직접 측정 가능. 질량($m$), 감쇠 상수($c$), 강성($k$)이 해당됨.
- **Parameter (매개변수):** ODE 안에서 각 미분 차수 항을 연결하는 계수 역할. 시스템 물성이 수학적 표현 속에 들어올 때 parameter라 부름.

> 💬 **Property vs Parameter** (질문)
> M, C, K는 property인가 parameter인가?
> → **둘 다다.** 물리 시스템에서 측정된 물성(property)이 ODE에서 계수로 작용할 때 parameter가 된다. Property는 시스템의 정적 특성이고, parameter는 수학적 모델 안에서의 역할이다.

**Parameterization**

주파수 응답 함수를 절대 주파수 $\omega$ 대신 정규화 주파수 $r = \omega/\omega_n$으로 표현할 때, $\omega_n = \sqrt{k/m}$이므로 시스템 물성이 자연스럽게 내재된다. 이처럼 시스템의 물성 정보를 반영한 무차원 척도로 재표현하는 과정이 **parameterization**이다.

$$r = \frac{\omega}{\omega_n}, \quad \omega_n = \sqrt{\frac{k}{m}}$$

## 2. Parameter Excitation — 비선형 ODE 분류

**Duffing 방정식 (비선형)**

$$m\ddot{x} + c\dot{x} + kx + \beta x^3 = F(t)$$

$x^3$ 항 때문에 비선형. 대표적인 하드닝/소프트닝 스프링 모델.

**Mathieu 방정식 (Linear Time-Variant, LTV)**

$$m\ddot{x} + c\dot{x} + k(t)\,x = F(t) \quad \text{또는} \quad k(x)\,x = F(t)$$

$k(t)$처럼 계수가 시간에 따라 변하면 선형이지만 **시변 시스템(LTV)**이 된다.

**왜 이 식인가?**

실제 기어 쌍(gear pair)에서 접촉 점수가 1개 ↔ 2개로 교번하면 강성 값이 주기적으로 변한다. 이때 강성 자체가 excitation 원으로 작용하는데, 이를 **parameter excitation**이라 한다. 파라미터(여기서는 $k$)가 시간에 따라 변동하면서 시스템에 일종의 진동 입력을 만드는 현상이다.

> 💬 **시변 계수의 예** (질문)
> 기계공학 현장에서 LTV 조건이 발생하는 실제 부품은?
> → **기어(gear pair).** 이 접촉 반경이 변하면서 강성 $k(t)$가 주기적으로 진동 → parameter excitation 발생.

## 3. Distributed Parameter System → Lumped 근사

**개념 제시**

분포 매개변수 시스템(예: 연속 보, 로프)은 시간 $t$와 공간 $x$ 두 변수가 모두 포함된 **PDE**로 기술된다.

$$c^2 u_{xx} = u_{tt}, \quad c^2 u'' = \ddot{u}$$

**Variable Separation (변수분리)**

$$u(x,t) = X(x)\,T(t)$$

시간 함수와 공간 함수를 분리하면 두 개의 ODE로 분해 가능하다.

**Lumped 근사 (FEM 개념)**

연속체를 $m_1, m_2, \ldots, m_n$ 등 유한 개의 집중질량으로 이산화하면, PDE 문제가 ODE 시스템(Lumped Parameter System)으로 변환된다. 요소 수 $n$이 클수록 정확도 증가 → FEM의 기본 철학.

$$\text{PDE} \xrightarrow{\text{이산화 (FEM)}} \text{ODE 시스템 (Lumped)}$$

> 💬 **이산화 요소 수** (질문)
> 정확도를 높이려면 요소(mass segment)를 몇 개로 나눠야 하는가?
> → 이론적으로 $n \to \infty$일수록 연속체에 수렴. FEM은 이 철학을 체계화한 방법론이다.

## 4. RL 전기 회로 — First Order System 완전 분석

### 4-1. 기본 방정식 유도

**시스템 구성:** 전압원 $e_s(t)$ — 저항 $R$ — 인덕터 $L$ — 접지

**변수 정의:**

$$i_R = i_L = i, \quad i_R = \frac{e_R}{R} = \frac{e_s - e_A}{R}, \quad e_L = e_A = L\frac{di}{dt}$$

**유도 과정**

출발: KVL 및 소자 특성 적용

$$e_s - e_A = iR \quad \cdots (1)$$

$$e_A = L\frac{di}{dt} \quad \cdots (2)$$

↓ (2)를 (1)에 대입

$$e_s - L\frac{di}{dt} = iR$$

↓ 양변을 $R$로 나누고 정리

$$\frac{L}{R}\frac{di}{dt} + i(t) = \frac{1}{R}e_s(t)$$

결론:

$$\boxed{\tau\frac{di}{dt} + i(t) = \frac{1}{R}e_s(t), \quad \tau = \frac{L}{R}}$$

> 💬 **Time Constant** (질문)
> $\tau = L/R$의 물리적 의미는?
> → **시간 상수(time constant).** 입력이 step으로 인가된 후, 최종값의 **63%**에 도달하는 시간. $\tau$가 작을수록 더 빠르게 응답. 1차 시스템의 동특성을 결정하는 유일한 파라미터.

### 4-2. 전달 함수 (Laplace 변환)

Zero initial conditions 하에서 미분 연산자 $D \to s$ 치환:

$$(\tau s + 1)\,I(s) = \frac{1}{R}E_s(s)$$

$$\boxed{H(s) = \frac{I(s)}{E_s(s)} = \frac{1/R}{\tau s + 1}}$$

### 4-3. 시간영역 해법 — 동차해 + 특수해

**[방법 A] 시간영역 (동차해 + 특수해)**

$e_s(t) = 1$ (단위 계단 입력) 조건에서:

| 구분 | 풀이 |
|------|------|
| 동차해 | $i_h(t) = Xe^{st}$ 대입 → $(\tau s+1)Xe^{st}=0$ → $s = -1/\tau$ → $i_h = Xe^{-t/\tau}$ |
| 특수해 | $i_p = b_0$ (상수) → $\tau \cdot 0 + b_0 = 1/R$ → $b_0 = 1/R$ |
| 초기조건 | $i(0)=0$: $X + 1/R = 0$ → $X = -1/R$ |

$$\boxed{i(t) = \frac{1}{R}\left(1 - e^{-t/\tau}\right)}$$

**[방법 B] Laplace 변환 (검증)**

$E_s(s) = 1/s$ (단위 계단의 Laplace):

$$I(s) = \frac{1}{R} \cdot \frac{1}{s(\tau s+1)} = \frac{1}{R}\left(\frac{1}{s} - \frac{\tau}{\tau s+1}\right) = \frac{1}{R}\left(\frac{1}{s} - \frac{1}{s+1/\tau}\right)$$

역 Laplace 변환:

$$i(t) = \frac{1}{R}\left(1 - e^{-t/\tau}\right) \quad \checkmark \text{ 일치}$$

**검증 (극한 확인)**

- $t=0$: $i(0) = 0$ ✓ (인덕터는 전류 급변 불가)
- $t \to \infty$: $i \to 1/R$ ✓ (인덕터 → 단락, $i = e_s/R$)
- $t=\tau$: $i(\tau) = \frac{1}{R}(1-e^{-1}) \approx \frac{0.632}{R}$ ✓ (63% 도달)

## 5. 회전 시스템 — First Order로의 변환

**시스템 구성:** 관성모멘트 $J$, 점성감쇠 $b$, 입력 토크 $T_E(t)$

**뉴턴 제2법칙 (회전):**

$$J\ddot{\theta}(t) + b\dot{\theta}(t) = T_E(t)$$

**$\omega = \dot{\theta}$ 치환으로 First Order 변환:**

$$\boxed{J\dot{\omega}(t) + b\omega(t) = T_E(t)}$$

**왜 이 식인가?**

전기 시스템의 $\tau \frac{di}{dt} + i = \frac{1}{R}e_s$와 구조가 동일하다. $J \leftrightarrow \tau$, $b \leftrightarrow 1$, $\omega \leftrightarrow i$의 대응 관계 → **기계-전기 유추(analogy)** 의 핵심.

> 💬 **변수 치환의 이유** (질문)
> 왜 $\theta$ 대신 $\omega$를 상태변수로 쓰는가?
> → $\theta$를 상태변수로 두면 2차 ODE지만, $\omega = \dot{\theta}$로 치환하면 1차 ODE로 줄어 분석이 단순해진다. 전기계의 전류 $i$가 전하량 $q$의 미분인 것과 완전히 대응.

## 6. 전기-기계 연성 시스템 (Electro-Mechanical System) — 예고

**모터 내부의 두 가지 상수:**

$$T_D = K_T \cdot i \quad \text{(발생 토크)}$$

$$e_m = K_E \cdot \omega \quad \text{(역기전력, back-EMF)}$$

$K_T \approx K_E \equiv K$로 근사하면 두 시스템이 연결된다.

**결합 구조:**

- 전기계 ODE: $L\frac{di}{dt} + Ri + K\omega = e_s$
- 기계계 ODE: $J\dot{\omega} + b\omega = Ki$

→ $i$와 $\omega$가 서로 영향을 주는 **coupled system** (다음 주 상세 분석 예정)

> 💬 **모터와 발전기의 차이** (질문)
> 같은 조립체가 어떻게 모터도 되고 발전기도 되는가?
> → **에너지 흐름 방향만 다르다.** 전기 → 기계이면 모터, 기계 → 전기이면 발전기. 내부 구조(전기자, 자기장)는 동일하다.
